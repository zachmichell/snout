# 6.6a — Processor payouts (shipped end-to-end)

## What's live

End-to-end automation from Stripe to QBO:

```
Stripe Connect account
   │  (cron at 03:15 UTC, or "Pull from Stripe" button)
   ▼
quickbooks-ingest-stripe-payouts
   │  pulls payouts via Stripe Payouts API
   │  resolves charges -> payment_intent -> Snout payment
   ▼
processor_payouts (state='ready') + payment_payouts junction
   │  (cron at 03:15 UTC, or "Sync payouts now" button)
   ▼
quickbooks-sync-payouts
   │  posts a Journal Entry per payout
   ▼
QBO Journal Entry (mapped in quickbooks_entity_mappings)
```

## Why Journal Entry, not Bank Deposit

We initially tried the QBO Bank Deposit endpoint. It rejected every
payload variant we tried against the Canadian sandbox with `code 6000`
("ValidationFault") and `Detail: "Business Validation Error: Select a
bank account for this deposit."` plus an empty `element` field. We
verified through QBO's own Account API that the Bank account was
valid (CAD, active, not a sub-account, $21k balance) — there was no
body shape that QBO would accept.

A Journal Entry achieves the identical accounting effect and works
without issue. The double-entry posting per payout:

| Posting | Account | Amount |
|---|---|---|
| Debit | Operator's bank account (e.g. Chequing) | net = gross − fee |
| Debit | Fee expense account | fee |
| Credit | Undeposited Funds | gross |

Net effect on the GL is identical to a Bank Deposit pulling from
Undeposited Funds with a negative fee line. Reconciliation tools and
the bank register show the JE the same way.

## Operator setup

1. **Connect Stripe** (already done as part of payment processing).
2. **Connect QuickBooks** (already done as part of cluster 6).
3. **Pick the Fee account** in the QuickBooks settings tab → PayoutsCard.
   Dropdown shows your QBO Expense + COGS accounts; pick the one your
   accountant tracks merchant fees in (commonly "Merchant Processing
   Fees" or "Bank Service Charges").
4. **Pick a Bank account** — done by the same flow that picked the
   default deposit-to (Undeposited Funds). The `default_bank_account_id`
   column needs to be populated; if it's null, the sync errors with a
   clear message and the operator picks one in QBO.
5. **First-time backfill**: click "Pull from Stripe" — pulls payouts
   from the past 30 days. Then click "Sync payouts now" to post the
   journal entries to QBO.
6. **Steady state**: the cron at 03:15 UTC ingests + syncs nightly.
