# 6.6a — Bank Deposit shape follow-up

## What's shipped

- `processor_payouts` + `payment_payouts` tables with RLS
- `quickbooks_accounts.default_fee_account_id`, `default_fee_account_name`,
  `default_bank_account_id`, `default_bank_account_name`

  `quickbooks-sync-payouts`

- QBO REST helpers: `createDeposit`, `updateDeposit`, `listExpenseAccounts`

## Open issue

Bank Deposit creation against the Canadian sandbox returns:

> Business Validation Error: Select a bank account for this deposit.

Even though `DepositToAccountRef.value = '38'` (Chequing, AccountType=Bank).

Two suspected causes (need debugging against the operator's real
realm or with QBO's request inspector):

1. **AccountRef shape**: QBO may want `value` as a number, not string, in
   some cases. Try `Number(account.default_bank_account_id)`.
2. **Required `name` field on the account ref**: try sending `name`
   matching the QBO display name exactly. We currently send "Chequing"
   but the realm may have it stored as "Chequing - CAD" or similar.

The line shape `DepositLineDetail.AccountRef` for the source account
may also need adjustment. QBO docs hint that for AccountRef-based
lines (no LinkedTxn), the source account should be the SAME currency
as the DepositToAccountRef.

## Workaround for now

Operators can manually insert `processor_payouts` rows via SQL after
matching their bank statement, then click "Sync payouts now". Once the
Deposit shape is fixed, this same row will sync without re-insert.
